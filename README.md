# 복약 체크 (Pill Tracker)

매일 먹는 약을 기록하고, 남은 수량을 관리하는 아이폰용 PWA(웹앱)입니다.
모든 데이터는 기기(브라우저) 안에만 저장되며 서버·로그인이 필요 없습니다.

## 기능

- **오늘 탭** — 시간대별(아침/점심/저녁/취침 전) 복용 체크리스트. 체크하면 재고가 자동 차감되고, 체크 해제하면 복구됩니다.
- **내 약 탭** — 약 등록/수정/삭제, 재고 추가. 설정한 일수 이하로 남으면 경고 표시.
- **기록 탭** — 달력에서 날짜별 복용 현황 확인 (모두 복용 🟢 / 일부 🟡 / 안 먹음 🔴).

## 파일 구성

| 파일 | 역할 |
|---|---|
| `index.html` | 앱 뼈대 |
| `style.css` | 디자인 (모바일 우선) |
| `app.js` | 앱 로직 전체 (데이터는 localStorage에 저장) |
| `manifest.webmanifest` | 홈 화면 설치용 PWA 설정 |
| `sw.js` | 오프라인 지원 서비스 워커 |
| `cloud.js` | (선택) Firebase 클라우드 동기화 모듈 |
| `icons/` | 앱 아이콘 |

## PC에서 미리 보기

```powershell
cd pill-tracker
npx serve .
```

또는 `index.html`을 브라우저로 바로 열어도 됩니다(서비스 워커만 비활성).

## 아이폰에 설치하기

1. 이 폴더를 **GitHub Pages** 또는 **Vercel**에 무료 배포 (HTTPS 주소 필요)
2. 아이폰 **Safari**로 해당 주소 접속
3. 공유 버튼 → **"홈 화면에 추가"**
4. 홈 화면 아이콘으로 실행하면 일반 앱처럼 전체 화면으로 동작

> ⚠️ 데이터는 기기에 저장되므로, Safari 설정에서 "방문 기록 및 웹 사이트 데이터 지우기"를 하면 기록이 사라집니다. (홈 화면 앱의 데이터는 별도 보관되어 비교적 안전합니다.)

## 데이터 보관 방식: 로컬 vs 클라우드

**내 약** 탭의 **클라우드 동기화** 카드에서 두 방식 중 하나를 직접 선택합니다.

- **로컬 전용 (기본값)** — 데이터가 이 기기에만 저장됩니다. 가장 단순하고 빠르며, 외부로 아무것도 나가지 않습니다. 클라우드를 켜기 전까지 Firebase 코드는 아예 로드되지 않습니다.
- **클라우드 동기화 (선택)** — 약을 체크할 때마다 Firebase(Firestore)에 자동 저장되고, 같은 동기화 코드를 입력한 다른 기기와 실시간으로 공유됩니다. 백업 공백이 없습니다.

### 클라우드 동기화 설정 (한 번만)

무료 Firebase 프로젝트가 필요합니다. (Gmail 계정이면 별도 가입 없이 가능)

1. [Firebase 콘솔](https://console.firebase.google.com) → **프로젝트 추가**
2. 프로젝트 안에서 **웹 앱**(`</>`) 추가 → 표시되는 `firebaseConfig` 값을 복사
3. 좌측 메뉴 **빌드 → Firestore Database** → **데이터베이스 만들기** (프로덕션 모드)
4. **빌드 → Authentication → 로그인 방법** → **익명(Anonymous)** 사용 설정
5. Firestore의 **규칙(Rules)** 탭에 아래를 붙여넣고 게시:

   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /syncs/{code} {
         allow read, write: if request.auth != null;
       }
     }
   }
   ```

6. 웹앱의 **내 약 → 클라우드 동기화 설정**에서 복사한 `firebaseConfig`를 붙여넣고 **연결**
7. 표시되는 **동기화 코드**를 메모 (다른 기기에서 같은 기록을 보려면 그 기기에 이 코드를 입력)

### 다른 기기에서 같은 기록 보기

새 기기의 **클라우드 동기화 설정**에서 같은 `firebaseConfig`를 넣고, **기존 동기화 코드** 칸에 첫 기기에서 받은 코드를 입력하면 기록이 그대로 따라옵니다.

> 🔐 **보안 메모:** 동기화 코드는 비밀번호처럼 다루세요. 이 코드를 아는 사람은 기록을 볼 수 있습니다(32자리 랜덤이라 추측은 사실상 불가능). Firebase 설정값(apiKey 등)은 원래 웹앱에 공개되는 값이라 노출돼도 위 보안 규칙이 무단 접근을 막습니다.

## 알림 설정 (아이폰 단축어 자동화)

앱 자체 푸시 대신 아이폰 기본 자동화를 사용합니다.

1. **단축어** 앱 → **자동화** 탭 → **+**
2. **특정 시간** 선택 (예: 오전 8시, 매일) → "즉시 실행" 켜기
3. 동작 추가: **"알림 표시"** → 내용 예: "아침 약 드실 시간이에요 💊"
4. 저녁 등 다른 복용 시간도 같은 방식으로 추가

복용 시간에 알림을 받으면 → 홈 화면의 **복약 체크** 앱을 열어 체크하면 됩니다.

> 💡 **알림을 누르면 앱이 바로 열리게 하려면:** 자동화 동작에서 **"앱 열기"**(URL 열기 ❌)를 추가하고 앱 목록에서 **"복약 체크"**를 선택하세요(iOS 16.4 이상). **"URL 열기"는 Safari 새 탭으로 열려 기록이 빈 화면이 나오므로 쓰면 안 됩니다** — 홈 화면 앱과 Safari 탭은 저장 공간이 분리되어 있습니다.

## 파일 수정 후 배포 시 주의

`sw.js`의 `CACHE_VERSION` 값을 올려야(예: `v3` → `v4`) 이미 설치한 기기에 새 버전이 반영됩니다.
